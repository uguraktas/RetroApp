## Overview

CodeBaseHub is highly configurable to fit your specific needs. This guide covers all configuration options.

---

## Environment Variables

All environment variables should be in a `.env` file at the project root.

> **Warning:** Never commit your `.env` file to version control.

### Required Variables

```bash
# Database
DATABASE_URL="postgresql://user:password@localhost:5432/codebasehub"

# Authentication
AUTH_SECRET="your-super-secret-key-min-32-chars"
AUTH_TRUST_HOST="true"

# Application
NEXT_PUBLIC_APP_URL="http://localhost:3001"
```

### Optional OAuth Variables

```bash
# GitHub OAuth
GITHUB_CLIENT_ID="your-github-client-id"
GITHUB_CLIENT_SECRET="your-github-client-secret"

# Google OAuth
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"
```

## Application Configuration

The main configuration is in `packages/config/index.ts`:

```typescript
export const config = {
  appName: "CodeBaseHub",
  description: "A production-ready SaaS starter kit",
  author: "Your Name",
  
  i18n: {
    default: "en",
    locales: {
      en: "English",
      tr: "Türkçe",
      ar: "العربية",
    },
  },
};
```

## Theme Configuration

Customize colors in `tailwind.config.ts`:

```typescript
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: {
          DEFAULT: "#3b82f6",
          foreground: "#ffffff",
        },
      },
    },
  },
};
```

## Database Configuration

### Connection String

```bash
# PostgreSQL (recommended)
DATABASE_URL="postgresql://user:password@localhost:5432/dbname"

# MySQL
DATABASE_URL="mysql://user:password@localhost:3306/dbname"
```

### Drizzle Configuration

```typescript
import { defineConfig } from "drizzle-kit";

export default defineConfig({
  schema: "./apps/web/src/lib/db/schema.ts",
  out: "./apps/web/drizzle",
  driver: "pg",
  dbCredentials: {
    connectionString: process.env.DATABASE_URL!,
  },
});
```

## Internationalization

### Adding a New Language

1. Update config in `packages/config/index.ts`
2. Create `apps/web/messages/{locale}.json`
3. Add translations:

```json
{
  "common": {
    "welcome": "Welcome",
    "signIn": "Sign In"
  }
}
```
